# 《helloAgent》——Agent 框架实现学习

《hello agent》在此章节中介绍了一个完整的智能体的实现——基础对话、工具调用。

---
## SimpleAgent

一个智能体最基本的就是对话。智能体对话采用的是大模型 api 调用，而 api 调用是无状态的，为了实现连续对话，需要实现一个历史记录结构，用来记录对话记录。历史记录结构本质就是一个字符串列表，每次 api 调用都将用户的提问和大模型返回的响应添加都列表中。

工具调用可以理解为编程上常用的“HelpFunction”——给大模型的帮助函数，所以我们需要有工具注册函数，负责将工具注册到全局结构——工具箱中。

有了代码的实现，我们也需要让大模型“知道”。我们需要将提示词组装好，包括系统提示词。

---
除了基本的智能体外，《HelloAgent》还介绍了 React、Reflection、PlanAndSolve 范式。
这三范式，我个人的理解是在基础智能体的实现上，设计了各自的一个工作流程。

## ReAct

```
MY_REACT_PROMPT = """你是一个具备推理和行动能力的AI助手。你可以通过思考分析问题，然后调用合适的工具来获取信息，最终给出准确的答案。

## 可用工具
{tools}

## 工作流程
请严格按照以下格式进行回应，每次只能执行一个步骤:

Thought: 分析当前问题，思考需要什么信息或采取什么行动。
Action: 选择一个行动，格式必须是以下之一:
- `{{tool_name}}[{{tool_input}}]` - 调用指定工具
- `Finish[最终答案]` - 当你有足够信息给出最终答案时

## 重要提醒
1. 每次回应必须包含Thought和Action两部分
2. 工具调用的格式必须严格遵循:工具名[参数]
3. 只有当你确信有足够信息回答问题时，才使用Finish
4. 如果工具返回的信息不够，继续使用其他工具或相同工具的不同参数

## 当前任务
**Question:** {question}

## 执行历史
{history}

现在开始你的推理和行动:
"""
```

## Reflection
```
DEFAULT_PROMPTS = {
    "initial": """
请根据以下要求完成任务:

任务: {task}

请提供一个完整、准确的回答。
""",
    "reflect": """
请仔细审查以下回答，并找出可能的问题或改进空间:

# 原始任务:
{task}

# 当前回答:
{content}

请分析这个回答的质量，指出不足之处，并提出具体的改进建议。
如果回答已经很好，请回答"无需改进"。
""",
    "refine": """
请根据反馈意见改进你的回答:

# 原始任务:
{task}

# 上一轮回答:
{last_attempt}

# 反馈意见:
{feedback}

请提供一个改进后的回答。
"""
}
```
